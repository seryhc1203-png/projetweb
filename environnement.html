Dans ce projet, nous allons explorer différents environnements d'algorithmes utilisés dans le domaine de l'intelligence artificielle. Ces environnements sont essentiels pour tester et évaluer les performances des algorithmes dans des situations réelles ou simulées. Nous allons notamment nous pencher sur les environnements de jeux, de labyrinthes, et d'apprentissage automatique.

Environnements du projet

Dans notre application, chaque algorithme d'Intelligence artificielle s'exécute dans un environnement spécifique. Un environnement correspond à un ensemble de données décrivant l'état du problème (grille, plateau, parterre…). Les algorithmes utilisent ensuite ces données pour calculer une solution, sans que la logique de calcul soit incluse dans les classes de données. Cette séparation permet de maintenir une architecture orientée objet claire et modulaire.

Environnement A* : le labyrinthe

L'environnement de A* est un labyrinthe représenté par une grille de cellules. Chaque cellule possède une position (x, y) et un état indiquant si elle est un obstacle ou non. Deux cellules particulières sont définies : la cellule de départ et la cellule d'arrivée.

  • Labyrinthe : contient la grille, le départ et l'arrivée.
  • Cellule : décrit une case de la grille (coordonnées + obstacle).

Environnement MinMax : le plateau de jeu

L'environnement de MinMax est un plateau de jeu 4*4 composé de cases. Les joueurs placent des nombres dans les cases, et l'objectif est d'obtenir un alignement respectant la règle du jeu (ex. somme paire). Le modèle de données stocke le plateau, les deux joueurs, et la liste des nombres disponibles.

  • JeuMinMax : représente une partie (joueurs, plateau, nombres disponibles).
  • Plateau : contient une grille de cases (4*4).
  • Case : peut contenir 0 ou 1 nombre.
  • Nombre : valeur entière manipulée par le jeu.
  • Joueur : identifiant + nom.

Environnement Perceptron : le parterre de roses

L'environnement du perceptron est un parterre contenant des roses. Chaque rose est décrite par des caractéristiques (ex. taille, intensité du rouge) et une position pour l'affichage. Le système sélectionne certaines roses pour constituer un bouquet.

  • ParterreRose : contient une liste de roses.
  • Rose : caractéristiques (taille, rouge) + position.
  • Bouquet : liste de roses sélectionnées + capacité maximale.
  • Perceptron : paramètres du modèle (poids, biais) stockés comme données.
