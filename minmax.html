L'algorithme MinMax est une technique utilisée dans les jeux à deux joueurs pour déterminer le meilleur coup à jouer. Il fonctionne en évaluant les différentes possibilités de jeu et en choisissant le coup qui maximise les chances de gagner pour le joueur tout en minimisant les chances de gagner pour l'adversaire.

Principe général

MinMax est un algorithme de décision utilisé dans les jeux à deux joueurs. Il explore les coups possibles sous forme d'arbre : le joueur courant cherche à maximiser ses chances de gagner, tandis que l'adversaire est supposé jouer de manière optimale pour minimiser ce résultat. L'algorithme permet ainsi de choisir un coup considéré comme le meilleur dans une situation donnée.

Jeu et objectifs

Dans notre projet, MinMax s'applique à un plateau 4*4. Les joueurs placent des nombres dans les cases du plateau. L'objectif est d'obtenir un alignement respectant la règle définie (par exemple : aligner 4 nombres dont la somme est paire).

Données manipulées (modèle de données)

  • JeuMinMax : stocke les deux joueurs, le plateau et les nombres disponibles.
  • Plateau : grille 4*4 de cases.
  • Case : contient éventuellement un nombre (0..1).
  • Nombre : valeur entière placée dans une case.
  • Joueur : informations sur un joueur (id, nom).

Étapes générales

  1. Générer tous les coups possibles à partir de l'état actuel.
  2. Simuler récursivement les tours adverses et joueurs.
  3. Évaluer les états finaux avec une fonction de score.
  4. Choisir le coup maximisant le score du joueur.

Résultat attendu

À partir d'un état du plateau et des nombres restants, MinMax doit proposer un coup : choix d'une case et d'un nombre. Dans l'application, l'IHM permettra de visualiser le plateau et les choix effectués, éventuellement avec une animation des étapes de recherche.
